§ 1 · Our policy

Open by default.

Every paper accepted at a peer-reviewed venue ships with a public repository within 30 days of camera-ready. The repository contains: the code that produced the results, the data (or the loader for it), the evaluation scripts, the prompts (if any), and a one-command reproduce.sh that rebuilds the headline figure end-to-end.

Code is licensed MIT; data licensing follows the source (we are explicit per dataset). PhDs in the lab are first authors on the artefact repositories of their own papers; the PI is a contributor.

Reproducibility

One command, cold start

Every artefact includes reproduce.sh tested on a freshly-pulled image. If it doesn't run, it's a bug.

Versioning

Semver + tagged releases

The exact commit used in the paper is git-tagged with the venue + year (e.g., tse-2026).

License

MIT for code · CC-BY for data

Same defaults across the org. Per-paper overrides documented in each repo's LICENSE.

§ 2 · Contributing

Outside contributions welcome.

We accept pull requests, bug reports, and reproduction notes on every public repo. Substantial methodological contributions are credited as co-authors on follow-up papers; small fixes are credited in the changelog.

For larger collaborations (joint papers, benchmark contributions, replication studies), email Antonio directly — amastropaolo@wm.edu.
