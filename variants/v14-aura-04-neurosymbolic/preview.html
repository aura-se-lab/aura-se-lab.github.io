// 01 — mission · spec ⊢ prose

Mission

spec.aura// prose 
-- the lab, as a record value
record Lab where
  founded     : Year
  dept        : Dept
  director    : Person
  pillars     : Set Avenue
  mission     : Prop

aura : Lab
aura = {
  founded  = 2024,
  dept     = CS@WM,
  director = Mastropaolo,
  pillars  = { Explainability,
               Efficiency,
               MultiAgent,
               Neurosymbolic,
               TaskAware },
  mission  = ∀ x ∈ pillars,
             contributes(aura, x)
              ∧ responsible(x)
              ∧ understandable(x)
}
⊢

The AURA Lab, founded in Fall 2024, advances research at the intersection of efficiency, automation, and explainability in software engineering. Our mission is to design innovative and responsible AI-driven solutions that improve developer productivity while addressing key challenges of transparency and resource efficiency. Through rigorous methods, open collaboration, and a commitment to impactful work, the AURA Lab contributes practical advances in AI-supported software engineering for academia, industry, and society.

obligation: contributes(aura, x) for every pillar x
// 02 — research avenues

Research Avenues

spec.aura// prose 
-- five constructors; four primary + one supporting
type Avenue
  = Explainability
  | Efficiency
  | MultiAgent
  | Neurosymbolic
  | TaskAware   -- supporting

primary : Set Avenue
primary = { Explainability, Efficiency,
            MultiAgent, Neurosymbolic }

axiom supporting_iff :
  ∀ a : Avenue,
    supporting(a) ↔ a = TaskAware
⊢

Five constructors, four primary and one supporting. Each entry below pairs an Avenue constructor with its prose description. The supporting direction is Task-Aware Code Automation. 

Explainability : Avenue
  |= reveals(model, reasoning)
  |= artifacts = { rationales,
                   traces, tests }
  |= enables(dev, { inspect,
                       trust, debug })
⊢

[01] Explainability & Interpretability

Methods that reveal how code models reason — rationales, traces, tests — so developers can inspect, trust, and debug automation. 

Efficiency : Avenue
  |= techniques = { Quantization,
                    Distillation,
                    PEFT }
  |= ∀ m : Model,
       latency(m) ≤ τ
        ∧ memory(m)  ≤ μ
        ∧ cost(m)    ≤ κ
  |= reproducible(metrics)
⊢

[02] Efficiency & Resource-Aware Automation

Quantization, distillation, and PEFT to deliver low-latency, memory-light, cost-aware automation measured on real workloads with reproducible metrics. 

MultiAgent : Avenue
  |= roles = { Plan, Code,
                Test, Review }
  |= ∀ r ∈ roles,
       aligned(r, goal)
  |= coordinated(roles)
        ⇒ coherent(assistance)
⊢

[03] Multi-Agent LLMs for SE

Role-aligned agents that plan, code, test, and review, coordinated to produce coherent, end-to-end developer assistance. 

Neurosymbolic : Avenue
  |= pipeline = LLM ⊕ Analysis
                          ⊕ Constraints
  |= ∀ o : Output,
       checkable(o) ∧ interpretable(o)
  |= produces ⊆ { Fix, Test,
                      Design }
⊢

[04] Neurosymbolic AI for SE

Hybrid pipelines pairing LLMs with program analysis and constraints to produce interpretable, checkable fixes, tests, and designs. 

TaskAware : Avenue   -- supporting
  |= inputs  = { Issue, Diff, Trace }
  |= outputs = { Summary,
                   TODO, Plan }
  |= ∀ g : Guidance,
       actionable(g) ∧ repoAware(g)
  |= reduces(developer_toil)
⊢

[+1] Task-Aware Code Automation supporting

From issues, diffs, and traces to concise summaries, TODOs, and plans; models that reduce developer toil with actionable, repo-aware guidance.
